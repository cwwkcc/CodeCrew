**The Scenario:** මුහුදු මංකොල්ලකාරයෝ (Pirates) 5 දෙනෙක් ඉන්නවා (අපි ඔවුන්ව **P1, P2, P3, P4, P5** ලෙස හඳුන්වමු). P1 තමයි නායකයා, P2 දෙවැනියා, ඒ විදිහට අනුපිළිවෙළ තියෙන්නේ. මුන් පස්දෙනාට රන් කාසි 100ක් හමුවෙනවා.

මෙම කාසි බෙදාගන්නා නීතිය මෙන්න:

1. මුලින්ම ඉහළම ශ්‍රේණියේ සිටින Pirate (P1) කාසි බෙදිය යුතු ආකාරය ගැන යෝජනාවක් (**Proposal**) ඉදිරිපත් කළ යුතුයි.
    
2. ඉන්පසු සියලුම දෙනා (P1 ඇතුළුව) මේ යෝජනාවට පක්ෂව හෝ විපක්ෂව ඡන්දය දිය යුතුයි.
    
3. **50% ක් හෝ ඊට වැඩි** පිරිසක් පක්ෂ වුණොත්, ඒ යෝජනාව ක්‍රියාත්මක වෙනවා.
    
4. යෝජනාව පැරදුණොත් (ඡන්ද 50% ට වඩා අඩු වුණොත්), **P1 ව මරා දමනවා**. ඉන්පසු ඊළඟට සිටින නායකයා (P2) තම යෝජනාව ඉදිරිපත් කළ යුතුයි.
    

**The Pirates' Priorities:**

- මුලින්ම තමන්ගේ පණ බේරා ගැනීම.
    
- දෙවනුව තමන්ට ලැබෙන කාසි ප්‍රමාණය උපරිම කර ගැනීම.
    
- තමන්ට ලැබෙන කාසි ප්‍රමාණය සමාන නම්, අනිත් අයව මරා දැමීමට කැමැත්තක් දැක්වීම.
    

**The Challenge:** **P1** තම පණ බේරාගෙන, තමන්ට ලබාගත හැකි **උපරිම** කාසි ප්‍රමාණය ලබාගැනීමට ඉදිරිපත් කළ යුතු යෝජනාව කුමක්ද? (කාසි 100 බෙදෙන විදිහ P1, P2, P3, P4, P5 ලෙස ලියන්න).

**The Solution:** මේක පටලැවෙන්නෙ අපි මේක "ඉස්සරහට" (P1 ගේ පැත්තෙන්) හිතන්න යන නිසයි. මේ වගේ ප්‍රශ්නයක් විසඳන්න තියෙන ලේසිම ක්‍රමය තමයි **අන්තිමයාගේ ඉඳන් මුලට** හිතන එක. මේකට කියන්නේ **Backward Induction** කියලා.

#### **Case 1: දෙන්නෙක් ඉතිරි වුණොත් (P4, P5)**

- P4 තමයි යෝජනාව ඉදිරිපත් කරන්නේ.
    
- P4 තමන්ටම ඡන්දය දෙනවා (ඡන්ද 1/2 යි = 50%).
    
- නීතියට අනුව 50% ක් තිබ්බම යෝජනාව සම්මතයි.
    
- එතකොට P4 ගන්නවා **කාසි 100 ම**, P5 ට **0** යි.
    
- **ඉගෙනගත් පාඩම:** P5 දන්නවා අන්තිමට දෙන්නෙක් විතරක් ඉතිරි වුණොත් තමන්ට සතයක්වත් ලැබෙන්නේ නැහැ කියලා.
    

#### **Case 2: තුන්දෙනෙක් ඉතිරි වුණොත් (P3, P4, P5)**

- P3 දන්නවා P5 ගොඩක් බයයි කියලා (මොකද P3 පැරදිලා මැරුණොත් ඊළඟට වෙන්නේ Case 1, එතකොට P5 ට 0 යි).
    
- ඉතින් P3 යෝජනා කරනවා: **P3: 99, P4: 0, P5: 1**.
    
- P5 මේකට පක්ෂව ඡන්දය දෙනවා (මොකද 0 ට වඩා 1 ලොකුයි).
    
- ඡන්ද 2/3 කින් යෝජනාව සම්මතයි.
    

#### **Case 3: හතරදෙනෙක් ඉතිරි වුණොත් (P2, P3, P4, P5)**

- දැන් P2 යෝජනාව ඉදිරිපත් කරනවා. එයාට තව එක ඡන්දයයි ඕනේ (එයාගේ ඡන්දෙත් එක්ක 2/4 = 50% වෙන්න).
    
- P2 දන්නවා P3 ට කලින් Case එකේදී (Case 2) කාසි 99ක් ලැබෙන්න තිබුණා කියලා. ඒ නිසා P3 ව නමන්න අමාරුයි.
    
- හැබැයි P2 දන්නවා P4 ට කලින් Case එකේදී **0** ක් තමයි ලැබෙන්න තිබුණේ කියලා.
    
- ඉතින් P2 යෝජනා කරනවා: **P2: 99, P3: 0, P4: 1, P5: 0**.
    
- P4 පක්ෂව ඡන්දය දෙනවා (මොකද 0 ට වඩා 1 ලොකුයි).
    
- ඡන්ද 2/4 කින් යෝජනාව සම්මතයි.
    

#### **Case 4: පස්දෙනාම ඉන්නවා නම් (P1, P2, P3, P4, P5)**

- දැන් තමයි නියම සෙල්ලම. P1 ට තමන්ගේ ඡන්දෙට අමතරව තව ඡන්ද 2ක් ඕනේ (3/5 = 60%).
    
- P1 බලනවා P2 ගේ යෝජනාව (Case 3) සම්මත වුණොත් අනිත් අයට ලැබෙන්න තිබුණේ මොනවද කියලා:
    
    - P3 ට ලැබෙන්නේ: **0**
        
    - P4 ට ලැබෙන්නේ: **1**
        
    - P5 ට ලැබෙන්නේ: **0**
        
- P1 කරන්නේ මේ අතරින් අඩුවෙන්ම ලැබෙන්න ඉන්න දෙන්නාව "බිලී බාගන්න" එක. ඒ කියන්නේ **P3** සහ **P5**.
    
- P1 යෝජනා කරනවා: **P1: 98, P2: 0, P3: 1, P4: 0, P5: 1**.
    
- P3 සහ P5 දෙන්නම පක්ෂව ඡන්දය දෙනවා. මොකද P1 මැරිලා P2 ට අවස්ථාව ගියොත් ඒ දෙන්නටම ලැබෙන්නේ **0** කාසි ප්‍රමාණයක්.
    
- ඡන්ද 3/5 කින් යෝජනාව සම්මතයි!
    
    
## ප්‍රශ්නයේ තියෙන "තමන්ට ලැබෙන කාසි ප්‍රමාණය සමාන නම්, අනිත් අයව මරා දැමීමට කැමැත්තක් දැක්වීම." කියන වගන්තියෙන් කියන්නේ මොකද්ද??

මේක තමයි මංකොල්ලකාරයන්ගේ තියෙන **"Bloodthirsty"** ස්වභාවය. මේකෙන් අදහස් කරන්නේ සරලවම මෙන්න මෙහෙම දෙයක්:

හිතන්න ඔයා Pirate කෙනෙක් කියලා. ඔයාට යෝජනා දෙකක් එනවා:

1. **යෝජනාව A:** ඔයාට කාසි 5ක් ලැබෙනවා, නායකයා බේරෙනවා.
    
2. **යෝජනාව B:** ඔයාට කාසි 5ක් ලැබෙනවා, නායකයාව මරනවා.
    

මේ අවස්ථා දෙකේදීම ඔයාට ලැබෙන කාසි ගණන සමානයි (**5**). ඒ වගේ වෙලාවක මේ Pirates ලා හැමතිස්සෙම තෝරාගන්නේ **යෝජනාව B**. ඒ කියන්නේ තමන්ට අවාසියක් වෙන්නේ නැත්නම්, තවත් කෙනෙක්ව මරලා දාන්න (Eliminate කරන්න) ඔවුන් දෙපාරක් හිතන්නේ නැහැ.

### **ඇයි මේ නීතිය මෙතනට වැදගත් වෙන්නේ?**

මේ නීතිය නැති වුණොත්, P1 ට පුළුවන් නිකම්ම "මාව බේරගන්න, මම ඔයාට කලින් ලැබෙන්න තිබුණ ගාණම දෙන්නම්" කියලා මිනිස්සුන්ව බේරගන්න. හැබැයි මේ නීතිය නිසා **P1 ට සිද්ධ වෙනවා කලින් ලැබෙන්න තිබුණට වඩා "අඩුම තරමේ එක කාසියක් හරි වැඩියෙන්" දෙන්න.** **උදාහරණයක් විදිහට:**

- Case 3 එකේදී P5 ට එක කාසියක් (1) ලැබෙන්න තිබුණා.
    
- Case 4 එකේදී P2, P5 ට කාසි 0 ක් දුන්නොත්, P5 අනිවාර්යයෙන්ම "විරුද්ධව" ඡන්දය දෙනවා. මොකද P2 මැරුණොත් (Case 3 එනවා) P5 ට කොහොමත් 1ක් ලැබෙනවා.
    
- හැබැයි P2, P5 ට **එක කාසියක්ම (1)** දුන්නොත්? එතකොටත් P5 "විරුද්ධව" ඡන්දය දෙනවා!
    
- **හේතුව:** කාසි ගණන සමාන නිසා (1 = 1), P5 කැමතියි P2 ව මරා දාන්න.
    
- ඒ නිසා P2 ට සිද්ධ වෙනවා P5 ව ගන්න නම් කාසි **2ක්** දෙන්න. (නැත්නම් P5 ට 0 දෙන වෙනත් කෙනෙක්ව (P4 ව) කාසියකට බිලී බාගන්න).

#### **මෙමගින් Logic එකට වෙන බලපෑම:**

මේ නීතිය නිසා නායකයාට සිද්ධ වෙනවා අනිත් අයට **"තමන්ට ඊළඟ අවස්ථාවේදී ලැබෙන්න තියෙන ප්‍රමාණයට වඩා වැඩියෙන්"**යමක් දෙන්න. එහෙම නැති වුණොත් අනිත් අය ඡන්දය දෙන්නේ ඔහුව මරා දැමීමටයි.
මේක තමයි **Competitive Programming** වලදී **"Worst-case scenario"** එක විදිහට සලකන්නේ. හැමෝම ඔයාට විරුද්ධව ඉන්න වෙලාවක, ඔයා කොහොමද ඔයාගේ වාසිය හදාගන්නේ කියන එකයි මෙතන තියෙන නියම logic එක

## **මෙතන තියෙන Logic එක මොකක්ද?**

මේකට කියන්නේ **Game Theory** කියලා. මෙතනදී අපි උපකල්පනය කරන්නේ හැමෝම තමන්ගේ වාසිය විතරක් බලන, ඉතාම බුද්ධිමත් (Rational) පිරිසක් කියලා.

- **The Core Logic:** ඔයාට සහයෝගය ගන්න ඕනේ නම්, ඔයා යන්න ඕනේ ඔයාගේ යෝජනාව නැති වුණොත් **වැඩිපුරම අවාසියක් වෙන පුද්ගලයා** ගාවටයි. ඔහුට පොඩි වාසියක් දීලා ඔයාගේ පැත්තට ගන්න පුළුවන්.
    

මේක තමයි **Game Theory** කියන්නේ. ලොකු ලොකු **ව්‍යාපාරික තීරණ (Business Deals)**, **දේශපාලන උපාය මාර්ග (Political Strategies)** වගේ දේවල් වලදී මේ වගේ **Backward Induction** logic පාවිච්චි කරනවා.

- **Optimal Thinking:** පෙනෙන දේට වඩා ගැඹුරට හිතලා, ප්‍රතිවාදියාගේ "අවම වාසිය" තේරුම් අරගෙන, තමන්ට වාසිදායකම තීරණය ගන්න පුරුදු වීම තමයි **True Computational Thinking** කියන්නේ. මේක දියුණු කරගන්න එක ඔයාගේ අනාගතයට ඉතා වැදගත් වේවි!
